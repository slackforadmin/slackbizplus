주요 기능:<br>
Slack 워크스페이스의 보안 및 접근 권한을 관리하는 중앙 설정 화면이에요<br>
로그인 보안, 세션 관리, 비밀번호 정책 등 다양한 보안 옵션을 제어할 수 있어요<br>
설정 항목:<br>
로그인 설정: 이메일 로그인 2단계 인증, 필요하지 않음으로 설정되어 있으며 사용자가 이메일과 비밀번호만으로 로그인 가능해요<br>
세션 관리: 데스크톱 세션기간과 모바일 세션기간을 각각 설정할 수 있으며, 현재 비활성화된 상태예요<br>
비밀번호 강제 재설정: 이메일 주소 및 비밀번호로 로그인하는 멤버가 강제로 새 비밀번호를 생성하도록 할 수 있어요<br>
장치 설정: 관리되지 않는 장치에서 Slack 및 파일 액세스 설정을 구성할 수 있어요<br>
파일 다운로드: 데스크톱과 모바일 앱에서 파일 다운로드 허용 여부를 제어할 수 있어요<br>
각 설정 항목마다 "편집" 또는 "재설정" 버튼이 있어 손쉽게 보안 정책을 변경할 수 있어요!<br>
모바일 기기에서 메시지 복사하기 - 현재 "허용됨"으로 설정되어 있으며, 멤버들이 Slack 모바일 앱에서 메시지를 복사할 수 있어요. 데이터 유출 방지를 위해 필요시 이 기능을 비활성화할 수 있어요.<br>
탈옥 또는 루팅된 기기 - 현재 "허용됨"으로 설정되어 있어, 탈옥 또는 루팅된 기기를 사용하는 멤버도 Slack에 액세스할 수 있어요. 보안 강화를 위해 이러한 기기의 접근을 차단할 수도 있어요.<br>



로그인 설정
이메일 로그인의 2단계 인증
2FA 를 설정하는 옵션 입니다. 그리드에서는 문자 메세지는 사용할 수 없으며 2FA 앱 중에 선택할 수 있습니다.

그리드에서는 SAML 인증을 통해 SSO 로그인을  하는 경우에는 슬랙에서 2FA 를 설정하지 않고 IDP 에서 2FA 를 설정해야 합니다.

SSO 인증을 사용하지 않고 이메일/패스워드를 통해 로그인시 2FA 를 사용하도록 할 수 있습니다. 사용할 수 있는 2FA 앱은 다음과 같습니다.

iPhone: Google Authenticator, Duo Mobile, 1Password, Authy, Microsoft Authenticator
Android: Google Authenticator, Duo Mobile, 1Password, Authy, Microsoft Authenticator
데스크톱 세션 기간
기본값 : 비활성화 

설정시 데스크톱과, 모바일 별로 세션 기간을 지정할 수 있습니다.

모바일 세션 기간
기본값 : 비활성화 

설정시 데스크톱과, 모바일 별로 세션 기간을 지정할 수 있습니다.

비밀번호 강제 재설정
SSO 가 아닌 이메일/패스워드를 통해 로그인하는 멤버들의 비밀번호를 강제로 재설정하도록 할 수 있습니다.

장치 설정
멤버에 대한 EMM 앱 요구 사항
관리자가 자체적으로 설정을 변경할 수 없으며 모바일 디바이스에 대한 EMM (MDM) 사용시  설정 할 수 있습니다. EMM 사용시 Slack 에 연락 주십시요

게스트에 대한 EMM 앱 요구 사항
관리자가 자체적으로 설정을 변경할 수 없으며 모바일 디바이스에 대한 EMM (MDM) 사용시  설정 할 수 있습니다. EMM 사용시 Slack 에 연락 주십시요

파일 다운로드
기본값 : 데스크톱 / 모바일 허용

데스크톱 : 지정된 IP 에서 접속하는 경우에만 파일 다운로드를 할 수 있도록 설정할 수 있습니다. (이미지 미리보기의 경우 Blur 처리 됩니다) 지정가능한 IP 범위는 100 2000개 입니다.

모바일 : 모바일 기기에서 다운로드 하지 못하도록 차단할 수 있습니다.

모바일 기기에서 메세지 복사하기 
기본값 : 허용되지 않음

모바일 앱에서 복사를 차단합니다. 

모바일 비밀번호
모바일에서 슬랙앱 진입시 비밀번호를 설정하도록 강제 할 수 있습니다.   (EMM 미사용시만 적용)

선택가능한 옵션 : 즉시, 5분 후, 15분 후, 30분 후, 1시간 후

필수 모바일 브라우저 
모바일 디바이스에서는 슬랙 로그인시 브라우저를 경유해서 하게 되는데, 선택된 브라우저를 쓰도록 강제 하는 기능입니다.  (EMM 미사용시만 적용)
MS 엣지와 블랙베리 액세스 브라우저만 지원함으로 국내에서는 거의 사용되지 않는 기능입니다.

탈옥 기기나 고정된 장치 감지
기본값 : 비활성화

고정된 으로 번역되어 있으나 영어 원문은 Root 입니다. 따라서 탈옥된 디바이스(iOS) 나 루팅된 디바이스(Android) 인 경우 Slack 앱을 사용하지 못하도록 설정할 수 있습니다. (EMM 미사용시만 적용)

최소 앱 버전
기본값 : 비활성화

모바일 디바이스에 설치된 슬랙 앱의 최소버전을 지정할 수 있습니다. 현재 최신버전 포함 직전 5개의 버전까지 지정할 수 있으며, 그 이하 버전을 사용시에는 앱을 업데이트 하라는 문구가 발생하며, 업데이트 하기 전까지는 로그인할 수 없습니다. (EMM 미사용시만 적용)

EMM 을 사용하는 경우 EMM 설정을 우선 합니다.

이상 증후 이벤트 응답 설정
사용자 세션 자동 종료
기본값 :  2개의 이벤트 유형에 대해 활성화 됨 / 조직 주 소유자에게 전송된 알림

슬랙에서 정의하는 Anomaly Event 가 발생하는 경우에 자동으로 해당 사용자의 세션을 종료하거나, 또는 관리자/소유자에게 알람을 보낼 수 있도록 설정 할 수 있습니다.

이메일 알림 설정
초대된 멤버에게 메세지 내용 숨김 처리 
활성화 되는 경우 초대된 멤버가 아직 슬랙 가입을 하지 않은 경우에, 해당 멤버가 채널내에서 멘션 되거나, DM 을 받는 경우에 이메일로 해당 내용이 전송은 되지만, 실제 메세지는 가리는 기능 입니다. 이메일 도메인을 지정할 수 있습니다.

다음 문서를 참조하세요.

초대된 멤버용 TLS 시행
기본값 : 비활성화 

기본적으로 초대된 멤버에게 보내지는 이메일은 Opportunistic TLS 를 사용합니다. 이 말은 이메일을 받는 사용자가 TLS 를 사용하면, TLS 를 통해 보내고, 그렇지 않은 경우에는 non-TLS 로 보내집니다. 활성화시 TLS 를 사용해야만 이메일을 수신할 수 있습니다. 

Slack 에서 지원되지 않음
필수 업데이트 목록 받기
더이상 지원되지 않는 OS 나 앱 버전을 사용하는 멤버의 목록을 CSV 로 추출할 수 있습니다.
