
# 워크플로 단계, 트리거, 통합
조직내의 모든 워크플로 커넥터가 표시되며, 워크플로 승인/거부 여부를 결정할 수 있습니다.<br>

</div>

워크플로 트리거의 경우 워크플로 빌더에서 사용가능한 트리거는 6개 입니다. 해당 트리거의 사용 가능 여부를 정할 수 있습니다. (모든 사용자/특정 사람 및 워크스페이스/아무에게도 보내지 않음)

주요 구성 요소
워크플로 단계 (Workflow Steps)
워크플로 내에서 실행되는 개별 작업 단위
예: 양식 제출, 메시지 전송, 데이터 처리 등
타사 앱이 제공하는 커스텀 단계를 추가 가능

트리거 (Triggers)
워크플로를 자동으로 시작하는 이벤트
예: 특정 채널에 메시지 게시, 이모지 반응 추가, 예약된 시간 등
외부 앱이 제공하는 커스텀 트리거 사용 가능

통합 (Integrations)
외부 서비스와 Slack 워크플로를 연결
예: Google Sheets, Salesforce, Jira 등과의 데이터 연동
앱이 제공하는 API 기반 통합 기능

관리자 권한
워크스페이스에서 사용 가능한 커스텀 워크플로 기능 승인/차단
보안 및 규정 준수를 위한 통제
팀원들이 사용할 수 있는 자동화 도구 관리
이 설정을 통해 관리자는 조직의 업무 자동화 요구사항과 보안 정책 간의 균형을 유지할 수 있습니다!
