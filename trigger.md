
# 워크플로 단계, 트리거, 통합
조직내의 모든 워크플로 커넥터가 표시되며, 워크플로 승인/거부 여부를 결정할 수 있습니다.<br>

  <span style="color: #6A1B9A; font-weight: bold;">⚠️ 다만, 승인하더라도 사용자별로 해당 커넥터에 대한 인증은 개별적으로 진행해야 합니다.</span>
</div>

[주요 구성 요소]<br>
 - 워크플로 단계<br>
워크플로 내에서 실행되는 개별 작업 단위로, 타사 앱이 제공하는 커스텀 단계도 추가 가능합니다.

 - 트리거<br>
워크플로를 자동으로 시작하는 이벤트이며, 외부 앱이 제공하는 커스텀 트리거도 사용 가능합니다.

 - 통합<br>
외부 서비스와 Slack 워크플로를 연결할 수 있습니다.

 - 해당 트리거의 사용 가능 여부를 정할 수 있습니다. (모든 사용자/특정 사람/아무에게도 보내지 않음)
